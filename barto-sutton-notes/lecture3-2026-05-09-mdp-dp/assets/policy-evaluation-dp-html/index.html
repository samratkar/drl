Value Iteration

V(s) = maxa [r + γ V(s')]

At each sweep, every state takes the max over all actions — evaluation and improvement happen simultaneously. Convergence: max|ΔV| < 0.001

Comparison: Policy Iteration vs. Value Iteration

Key Insight: Both converge to the same V* and π*. Policy Iteration separates "evaluate" and "improve" cleanly. Value Iteration merges them — each Bellman update already takes the max, so it's both evaluating and improving at once. Value Iteration typically needs fewer total sweeps but each sweep does more work (max instead of weighted sum).

SARSA (On-Policy)

Q(s,a) ← Q(s,a) + α[r + γQ(s',a') - Q(s,a)]

Updates Q using the action actually taken (a') in the next state. Learns the value of the policy it follows. Convergence: max|ΔQ| < 0.01 after 50+ episodes

Ep: 0 | R: 0.0 ○ LEARNING

Q-Learning (Off-Policy)

Q(s,a) ← Q(s,a) + α[r + γ maxa' Q(s',a') - Q(s,a)]

Updates Q using the best possible action (max) in the next state. Learns optimal policy regardless of behavior. Convergence: max|ΔQ| < 0.01 after 50+ episodes

Ep: 0 | R: 0.0 ○ LEARNING
SARSA vs Q-Learning: SARSA uses the actual next action (a') — it learns the value of the ε-greedy policy it follows (safer, avoids danger). Q-Learning uses max — it learns the optimal policy regardless of exploration (bolder, may get closer to danger during learning).
