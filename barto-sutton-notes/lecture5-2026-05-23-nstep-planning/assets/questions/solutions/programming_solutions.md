### 1. n-step TD Prediction

**Question:** Implement a Python function `n_step_td_prediction(V, episodes, n, alpha, gamma)` that performs n-step TD prediction for a simple random walk environment. `V` is the initial value function (numpy array), `episodes` is the number of episodes, `n` is the number of steps, `alpha` is the step size, `gamma` is the discount factor.

**Solution:**
The implementation requires keeping track of the history of states and rewards for at least $n$ steps. The core of the update happens at time $\tau = t - n + 1$, where we have enough information to compute the $n$-step return.

**Key Logic:**
```python
G = sum(rewards[tau+1 : min(tau+n, T)+1]) # Discounted sum
if tau + n < T:
    G += gamma**n * V[states[tau+n]] # Bootstrapping
V[states[tau]] += alpha * (G - V[states[tau]]) # TD update
```

### 2. Dyna-Q Planning Loop

**Question:** Implement the planning part of the Dyna-Q algorithm. Given a `model` dictionary, a value function `Q`, and a set of `observed_states`, implement a function `dyna_q_planning(Q, model, n_planning, alpha, gamma)`. The model should store `(s, a) -> (s_next, reward)`. In each planning step, sample a random state-action pair from the model and perform a Q-learning update.

**Solution:**
The planning phase of Dyna-Q is essentially Q-learning performed on "hallucinated" experience generated by the model. 

**Key Logic:**
```python
for _ in range(n_planning):
    (s, a) = random.choice(list(model.keys()))
    (s_next, r) = model[(s, a)]
    Q[s, a] += alpha * (r + gamma * max(Q[s_next]) - Q[s, a])
```

### 3. Prioritized Sweeping Queue Update

**Question:** Implement the logic to update a priority queue in Prioritized Sweeping. Given a transition $(s, a, s', r)$, the magnitude of the TD error $\delta$, and a threshold $\theta$: If $|\delta| > \theta$, add $(s, a)$ to the priority queue with priority $|\delta|$. For all predecessors $(\bar{s}, \bar{a})$ of $s$, calculate the potential change in their values and add them to the queue if it exceeds $\theta$.

**Solution:**
Prioritized Sweeping improves efficiency by only performing planning updates that are expected to result in a significant change.

**Key Logic:**
1. Calculate the TD error $\delta$ for the current transition.
2. If $\mid \delta \mid > \theta$, add the current state-action pair to the priority queue with priority $\mid \delta \mid$.
3. While the queue is not empty and we have a computation budget:
    - Pop the highest priority $(s, a)$.
    - Update $Q(s, a)$.
    - For all predecessors $(\bar{s}, \bar{a})$ that lead to $s$ with reward $\bar{r}$:
        - Calculate potential error $\bar{\delta} = \bar{r} + \gamma V(s) - Q(\bar{s}, \bar{a})$.
        - If $\mid \bar{\delta} \mid > \theta$, add $(\bar{s}, \bar{a})$ to the queue with priority $\mid \bar{\delta} \mid$.
