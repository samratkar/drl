P[s][a] Data Structure

P[state][action] = [(probability, next_state, reward, is_terminal), ...]

This encodes the complete dynamics. Each tuple gives one possible outcome. Probabilities for a given (state, action) pair always sum to 1.0.

Policy Evaluation — Compute Vπ(s)

Given a fixed policy π, iteratively compute the value of each state using the Bellman expectation equation.

V(s) ← Σa π(a|s) · Σs',r p(s',r|s,a) · [r + γ·V(s')]

Asynchronous DP — Smarter Update Orders

Instead of sweeping all states in fixed order, update states intelligently based on priority or visitation.

Three Variants:
1. In-Place VI — Use updated values immediately (Gauss-Seidel style)
2. Prioritized Sweeping — Update states with largest Bellman error first
3. Real-Time DP — Only update states actually visited during episodes

Monte Carlo Prediction — Learn from Episodes

No model needed! Generate full episodes, compute returns, and average them to estimate Vπ(s).

V(s) ← V(s) + α · [Gt - V(s)] where Gt = rt+1 + γ·Gt+1

Learn optimal Q(s,a) directly by bootstrapping: use max Q(s',a') as the estimate of future value.

Q(s,a) ← Q(s,a) + α · [r + γ·maxa'Q(s',a') − Q(s,a)]
