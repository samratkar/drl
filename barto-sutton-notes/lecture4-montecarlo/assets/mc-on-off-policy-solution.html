Monte Carlo Methods – On-Policy & Off-Policy

Complete worked solution with step-by-step computation tables

Question

Consider an agent navigating through a simple episodic MDP with states S₀, S₁, S₂, and a terminal state ST. At each non-terminal state, the agent can take one of two actions: Left (L) or Right (R).

S₀
R, r=+1
S₁
L, r=+3
S₂
R, r=+5
ST

The agent generates the following episode under a behavior policy b (ε-greedy, ε=0.4):

Episode: S₀ →(R, r=+1)→ S₁ →(L, r=+3)→ S₂ →(R, r=+5)→ ST

Given:

Discount factor: γ = 0.9

Learning rate: α = 0.5 (for constant-α MC)

Initial Q-values: All Q(s, a) = 0

 

Behavior Policy b (ε-greedy, ε = 0.4):

At each state, the greedy action gets probability (1 − ε) + ε/2 = 0.6 + 0.2 = 0.8

The non-greedy action gets probability ε/2 = 0.2

 

Actions chosen by behavior policy in this episode:

S₀: R (greedy under b) → b(R|S₀) = 0.8

S₁: L (non-greedy under b) → b(L|S₁) = 0.2

S₂: R (greedy under b) → b(R|S₂) = 0.8

 

Target Policy π (deterministic greedy):

π(R|S₀) = 1.0, π(L|S₁) = 1.0, π(R|S₂) = 1.0

(For off-policy: the target policy always picks the action taken in this episode)

Find:

  1. Compute Q-value updates using On-Policy (first-visit) MC with constant-α
  2. Compute Q-value updates using Off-Policy MC with ordinary importance sampling
  3. Compare the results and explain the rationale for each approach
PART 1: ON-POLICY MC

On-Policy Monte Carlo (First-Visit, Constant-α)

Method

The agent follows one policy (the behavior policy b) and evaluates that same policy. No correction is needed since the data was generated by the policy we're evaluating.

Q(s,a) ← Q(s,a) + α [ Gt − Q(s,a) ]

Where Gt is the actual discounted return from time t until the end of the episode.

Step 1: Compute Returns from Each State-Action Pair

We work backwards from the terminal state to compute Gt at each time step.

PART 2: OFF-POLICY MC

Off-Policy Monte Carlo (Ordinary Importance Sampling)

Method

The agent follows the behavior policy b to collect data, but evaluates a different target policy π. Since the data comes from b but we want Qπ, we must correct for the distributional mismatch using importance sampling.

Q(s,a) ← Q(s,a) + α [ ρ · Gt − Q(s,a) ]

Where ρ is the importance sampling ratio — it re-weights the return to account for the probability difference between π and b:

ρt:T-1 = ∏k=tT-1 π(Ak|Sk) / b(Ak|Sk)

Why Are The Values Different?

The Core Issue: What Question Are We Answering?

Aspect On-Policy MC Off-Policy MC
Question answered "How good is action a at state s under policy b?" "How good is action a at state s under policy π?"
What it estimates Qb(s,a) — value under behavior Qπ(s,a) — value under target
Future behavior assumed Will keep exploring (ε=0.4) Will follow π perfectly (no exploration)
Why different? Exploration sometimes leads to suboptimal actions → lower value Assumes optimal actions always → higher value
Correction needed? No — data matches what we're evaluating Yes — importance sampling corrects mismatch

Rationale: When to Use Each

On-Policy MC — Use When:

  • Safety matters: Q-values reflect actual behavior including exploration mistakes (e.g., robotics, medical)
  • Simplicity: No importance sampling means no high-variance ratios
  • Policy is what you'll deploy: If you'll run the same ε-greedy policy in production
  • Exploration is part of the cost: Want Q-values that account for future exploratory actions

Trade-off: Can only learn about the current policy. To find the optimal policy, must slowly reduce ε (GLIE condition).

Off-Policy MC — Use When:

  • Want optimal policy: Learn Q* while exploring freely with any behavior policy
  • Reuse old data: Can learn from demonstrations, replay buffers, or other agents' data
  • Separate exploration from evaluation: Explore aggressively (high ε) while evaluating a greedy target
  • Multiple targets: Can evaluate many policies from a single stream of experience

Trade-off: Importance sampling can have very high variance. Long episodes → ρ products can explode or vanish, making learning unstable.

Why Does Importance Sampling Work?

Intuition

The behavior policy b generates episodes that are "wrong" for evaluating π. Some episodes happen more often under b than π (ρ < 1), and some happen less often (ρ > 1).

Importance sampling says: "If this episode would happen more often under π than b, it's more relevant — give it more weight. If it would happen less often, downweight it."

In This Problem

Time Action π prob b prob Ratio Meaning
t=0 R at S₀ 1.0 0.8 1.25 π would do this more often → upweight
t=1 L at S₁ 1.0 0.8 1.25 π would do this more often → upweight
t=2 R at S₂ 1.0 0.8 1.25 π would do this more often → upweight

Since π is deterministic and always chooses these exact actions, while b only does so with probability 0.8, every ratio is > 1. The cumulative ρ grows multiplicatively — this is why long episodes can cause variance issues.

The Variance Problem with Off-Policy MC

Why Ordinary IS Can Be Dangerous

Episode Length ρ product (if each ratio = 1.25) Effect on update
3 steps 1.25³ = 1.95 Return doubled — manageable
10 steps 1.25¹⁰ = 9.31 Return multiplied by 9 — very noisy
20 steps 1.25²⁰ = 86.7 Return multiplied by 87 — explosive
50 steps 1.25⁵⁰ = 8,882 Practically useless — one sample dominates all learning

Solutions to High Variance

Method How it helps Trade-off
Weighted IS Normalizes ratios: Σ(ρᵢGᵢ)/Σ(ρᵢ) — bounded Introduces bias (but consistent)
Per-decision IS Applies ratio only for relevant future steps, not all More complex to implement
Use TD instead Q-learning: off-policy without any IS needed! Introduces bootstrapping bias
Truncate ratios Cap ρ at some max value (e.g., 5) Introduces bias from capping

Key Takeaway: On-policy MC is simpler and more stable but can only learn about its own behavior. Off-policy MC can learn the optimal policy from exploratory data, but importance sampling introduces variance that grows exponentially with episode length. This is precisely why Q-learning (off-policy TD) is so popular — it achieves off-policy learning without importance sampling by bootstrapping with max Q(s',a').
